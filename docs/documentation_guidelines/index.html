ドキュメント作成のガイドライン

QGISドキュメントは https://www.qgis.org/resources/hub/#documentation にあります。執筆作業が進行している間、サポートされている全てのバージョン （テスト版、長期リリース (LTR)、LTRになる予定のもの）のビルドが毎日自動的に実行されます（正確な時刻はページの一番下を参照してください）。

QGIS Documentationのソースファイルは、 https://github.com/qgis/QGIS-Documentation で入手できます。これらは主に reStructuredText (reST) format の構文を使って書かれており、 Sphinx toolset のいくつかのスクリプトと結合して、HTML出力の後処理を行います。

以下の章では次のようなことを学ぶことができます。

  • ドキュメントのソースファイルを、git システムと、格納している GitHub プラットフォームを使って管理する方法

  • ルールに準拠した方法でテキストを変更し、スクリーンショットを提供する方法

  • あなたの行った変更を確実に公式ドキュメントに取り込み共有する方法

QGISプロジェクトに貢献する方法に関する一般的な情報を探しているなら、 QGISコミュニティに参加する にヘルプを見つけることができます。
